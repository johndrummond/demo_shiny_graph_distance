Overview

This is a sample shiny app illustrating a solution of the following problem.

Consider a graph with millions of nodes. These may be of two types U and P.

U nodes only connect to P nodes, if at all

P nodes only connect to U nodes

The weighing of all single connections is the same,

What is the distance between two U nodes where distance is defined as:

If the shortest path between two U nodes is via a single P node (i.e. they are both connected to the same P node) the distance is 1

If there is no path between two U nodes, the distance is 0

If the nodes to be checked are the same node the distance is defined as -1

The distance between two U nodes is always calculated based on the shortest path, meaning if U1 and U2 join directly to P1, they have a distance of one, moreover, if U2 and U3 join directly to P2, then the distance between U1 and U3 is 2, unless U1 and U3 both join directly to P3 (then itâ€™s 1 at that is the shortest path). U4 might join directly to P4 and P5, and therefore the distance between U1 and U4 is 0 as they are not connected.

Using the app

You can either enter your own graph data in the input box, or use the generator to create sample data.

There are five types of sample data graphs that can be calculated either a chain of u nodes, each connected to the next by one p node, or on of four random sample data sets where you can vary the level of connectivity.

The visualisation tab will show the network if there are fewer than 1000 nodes
